## フック

副業の案件が増えてくると、「手を動かす作業」より「確認する作業」のほうが地味に時間を食う。

クラウドワークスの新着チェック、納品前のテスト実行、作業ログの整理。どれも5分で終わるくせに、忘れると痛い。で、気づいたら1日に何回もターミナルを開いては同じコマンドを打ってた。

そんなときにClaude Codeに `/loop` っていう新機能が入った。要は「同じ指示を一定間隔で繰り返し実行してくれる」やつ。cronみたいなものがClaude Codeの中で動く。

これ、副業の「確認系タスク」と相性が良すぎた。

## この記事で得られること

- `/loop` コマンドの基本的な使い方
- 副業タスクを自動化する具体的なコマンド例3つ
- 使ってみて気づいた注意点と限界
- 「放置で回る仕組み」を作るコツ

## /loop ってなに？（30秒で分かる）

Claude Codeのターミナルで `/loop 10m やりたいこと` と打つと、10分ごとにその指示を自動実行してくれる機能。

```
/loop 10m エラーログをチェックして、新しいエラーがあれば報告して
```

これだけ。Claudeが間隔を解釈して、バックグラウンドでジョブを回してくれる。

ポイントは **セッションが開いている間だけ動く** こと。PCを閉じたら止まる。逆に言えば、「作業中にバックグラウンドで回しておく」使い方にぴったり。

間隔は `s`（秒）、`m`（分）、`h`（時間）、`d`（日）で指定できて、デフォルトは10分。最大3日で自動的に期限切れになるから、止め忘れても大丈夫。

## 使い方1: 新着案件の自動スキャン

自分はクラウドワークスとココナラで案件を受けてるんだけど、新着チェックって意外と面倒。条件に合う案件を探して、単価を確認して、応募するか判断して……これを1日3回くらいやってた。

で、こうした。

```
/loop 30m Webを検索して、クラウドワークスで「Python 自動化」「AI ツール開発」の新着案件をチェック。
単価5万円以上のものがあれば、案件名・URL・納期・単価をまとめて報告して。
```

30分ごとにClaudeが勝手に新着をチェックして、条件に合うものだけ報告してくれる。自分は他の作業をしながら、通知が来たら目を通すだけ。

**Before:** 1日3回 × 15分 = 45分
**After:** 通知を見るだけ。実質5分

## 使い方2: 納品前テストの定期実行

案件で作ったスクリプトを納品する前に、テストを何回か回しておきたい場面がある。特にAPI連携系は、相手のサーバーの状態で結果が変わることがあるから、1回通っただけだと不安。

```
/loop 15m python test_main.py を実行して、結果をまとめて。
FAILがあったらエラー内容を分析して、修正案も出して。
```

これで15分ごとにテストが走って、失敗があれば原因分析まで自動でやってくれる。寝る前にセットしておけば、朝起きたときに「全部PASSしてるよ」か「ここ直したほうがいいよ」が分かってる。

**Before:** テスト実行 → 結果確認 → 修正 → 再テスト（手動で繰り返し）
**After:** 放置で回して、朝に結果を確認するだけ

## 使い方3: 作業ログの自動生成

クライアントに「今日何やったか」を報告する必要がある案件が多い。これが地味に面倒で、作業に集中してると書き忘れる。

```
/loop 1h 直近のgit logとファイルの変更差分を確認して、
「何をやったか」「なぜやったか」「次にやること」の3点で作業ログを
work_log.md に追記して。時刻も入れて。
```

1時間ごとにgitの差分を見て、自動で作業ログを書いてくれる。納品時にこのログをそのまま送れるから、報告書を書く時間がほぼゼロになった。

**Before:** 作業後に30分かけて報告書作成
**After:** 自動生成されたログを軽く確認して送るだけ（5分）

## つまずきポイント

使ってみて気づいた注意点をまとめておく。

**セッションを閉じると止まる。** これが一番大事。/loopはClaude Codeのセッションが開いている間だけ動く。PCのスリープやターミナルを閉じたら消える。「24時間365日動かしたい」ならGitHub ActionsやOSのcronを使うべき。

**同時に50個まで。** 1セッションで動かせるジョブは最大50個。普通は5個も使えば十分だけど、テスト用に大量に作ると上限に当たる。

**最短間隔は1分。** 秒単位で指定もできるけど、実用上は1分以上にしたほうがいい。短すぎるとAPIコストが気になる。

**3日で自動消滅。** 止め忘れ防止のために3日の有効期限がある。長期で回すなら定期的にセットし直す必要がある。

## まとめ

/loopの本質は「確認と報告を自動化する」こと。

副業で一番もったいない時間は、「手を動かしてない時間」だと思う。案件を探してる時間、テスト結果を待ってる時間、報告書を書いてる時間。これらを/loopで自動化したら、自分は「判断する」と「作る」に集中できるようになった。

コマンド1行で始められるから、まずは `/loop 10m 何かチェックして` から試してみてほしい。

次回は、実際にクラウドワークスで受けた案件をClaude Codeでどう自動化したか、受注から納品までの全工程を書く予定。フォローしておくと通知が届きます。
