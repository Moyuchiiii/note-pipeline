結論: もう承認ボタンを連打しなくていい

Claude Codeに「Auto Mode」が追加された。3月24日リリース。

これまでClaude Codeを使ってると、ファイルを書き換えるたびに「許可しますか？」って聞かれてた。1回や2回ならいいけど、大きめのタスクを投げると10回、20回と承認ボタンを押すことになる。正直、これがめちゃくちゃ面倒だった。

Auto Modeは、そのAI自身が「この操作は安全か？」を判定して、安全なら勝手に進めてくれる機能。危険な操作だけブロックして、人間に確認を求める。

つまり、「全部手動で承認」と「全部スキップ（危険）」の間にある、ちょうどいいモード。

仕組み: AIが操作のリスクを自動判定する

Auto Modeの裏側では、各操作の実行前にAI分類器が動いている。

やってることはシンプルで、こういう判定をしている。

安全と判断される操作 → 自動で実行

プロジェクト内のファイルを読む
コードを編集する
テストを実行する

ブロックされる操作 → 人間に確認

ファイルの大量削除
機密データの外部送信
見知らぬスクリプトの実行

従来の「--dangerously-skip-permissions」フラグとの一番の違いは、プロンプトインジェクション対策が組み込まれていること。悪意のあるコードがClaude Codeを騙して危険な操作をさせようとしても、分類器がブロックしてくれる。

使い方: 2ステップで有効化

有効化は簡単。

ステップ1: ターミナルで以下を実行

claude --enable-auto-mode

ステップ2: セッション中に Shift + Tab で切り替え

Plan Mode → Auto Mode → Accept Edits → 通常モード の順で切り替わる。

VS Codeの拡張機能やClaude Desktopアプリからも使える。Settings → Claude Code から有効化して、セッション内のドロップダウンで選ぶだけ。

副業タスクで使ってみた

自分が一番「これはいい」と思ったのは、副業のコーディング案件で使ったとき。

クライアントから「CSVデータを整形して、グラフ付きのレポートを生成するスクリプトを作ってほしい」という依頼があった。Claude Codeに丸投げしたんだけど、従来モードだと途中で10回以上承認を求められていた。

Auto Modeで同じタスクを投げたら、ファイル作成・ライブラリのimport・テスト実行まで全部自動で進んでくれた。自分は途中で別の作業をしてて、戻ってきたら完成していた。

Before: タスク投げる → 承認 → 承認 → 承認…（15分張り付き）
After: タスク投げる → 放置 → 完成確認（実作業2分）

Auto Modeが最強な3つの場面

1. 大きめのリファクタリング

ファイルを10個以上編集するような作業で、いちいち承認していたら日が暮れる。Auto Modeなら投げて放置できる。

2. テスト→修正のループ

テストを走らせて、失敗したら修正して、また走らせて…というサイクルをAuto Modeなら自動で回してくれる。

3. /loopとの組み合わせ

前回紹介した/loop機能とAuto Modeを合わせると、定期的にコードチェック→自動修正まで完全放置でいける。

注意点

現在はリサーチプレビュー。Team planで利用可能、Enterprise/APIは近日対応
サンドボックス環境での使用が推奨されている
完全に安全ではない。重要なプロジェクトでは通常モードと使い分けるのが無難
Claude Sonnet 4.6 / Opus 4.6 で動作

まとめ

Auto Modeは「AIに任せたいけど、全部スキップするのは怖い」という人にとって、待ってました的な機能。

自分みたいに副業でClaude Codeを使ってる人にとっては、作業時間が体感で半分くらいになった。承認ボタンを押す時間って、意外とバカにならない。

次回は、Auto Modeと/loopを組み合わせて副業タスクを完全自動化してみた体験を書く予定。
