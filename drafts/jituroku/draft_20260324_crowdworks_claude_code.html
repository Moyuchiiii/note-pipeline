情報系の大学生がClaude Codeを相棒にして、クラウドワークスで実際に案件を受注して納品するまでの全工程を書く。

「AIで稼げる」って話はよく見るけど、実際にクラウドソーシングで案件取って、Claude Codeでコード書いて、クライアントに納品して、お金もらうところまでやった記録ってあんまりない。だから書く。

やったことの全体像

自分がやったのは、クラウドワークスで「業務自動化」系の案件を探して、Claude Codeで実装して納品するっていうシンプルな流れ。

ざっくり言うと：

  1. クラウドワークスで案件を探す
  2. 提案文を書いて応募する
  3. 受注したらClaude Codeで実装する
  4. テストして納品する
  5. お金をもらう

これだけ。でも、各ステップに「やってみたからわかるコツ」がある。

ステップ1: 案件の探し方

クラウドワークスで「自動化」「スクレイピング」「データ整理」「Python」あたりで検索する。

最初に気づいたのは、案件には大きく2種類あるってこと。

  • 「○○を自動化してほしい」系: 定型作業の自動化。スクレイピング、データ変換、レポート生成とか
  • 「○○のツールを作ってほしい」系: Webアプリやスクリプトの開発

自分が狙ったのは前者。理由は単純で、Claude Codeが一番得意な領域だから。

具体的に受注した案件の例（クライアント情報は匿名化してる）：

  • ECサイトの商品情報を毎日スクレイピングしてスプレッドシートに整理する仕組み
  • 複数のCSVファイルを統合してレポートを自動生成するスクリプト
  • 問い合わせフォームの内容をSlackに自動転送する連携

単価は1件あたり5,000〜30,000円くらい。大きい案件は取れないけど、小さい案件を数こなす作戦。

ステップ2: 提案文の書き方

ここが意外と大事だった。最初の5件くらいは全部落ちた。

落ちた提案文の共通点：

  • 「AIを使って効率的に開発します」→ クライアントはAIに抵抗ある人もいる
  • 「なんでもできます」→ 具体性ゼロ
  • テンプレ感丸出し

受注できるようになった提案文のポイント：

  • クライアントの課題を具体的に言い換える。「毎日手作業でコピペしてるの大変ですよね」みたいに
  • 納品物を明確にする。「Pythonスクリプト一式 + 使い方マニュアル + 動作テスト結果」
  • 納期を短めに出す。Claude Codeがあるから実際早い。「3日で納品します」は刺さる
  • 過去の実績（noteで書いた自動化記事のURL）を載せる

ちなみに「AIで開発します」とは書かない。クライアントが気にするのは「ちゃんと動くものが納期通りに届くか」であって、開発手法じゃない。

ステップ3: Claude Codeで実装する

ここが本題。受注した案件をClaude Codeでどう実装するか。

実際の流れ

まずクライアントからの要件を整理する。「○○サイトから商品名・価格・在庫状況を毎日取得して、Googleスプレッドシートに書き込んでほしい」みたいなやつ。

これをClaude Codeに投げる。ただし、丸投げはしない。

自分がやってるのはこんな感じ：

  1. 要件をClaude Codeに伝えて、まず設計を出してもらう
  2. 設計をレビューして、方針を決める
  3. Claude Codeにコードを書いてもらう
  4. 自分でテストして、動かないところを修正させる
  5. エラーハンドリングとドキュメントを追加させる

ポイントは②。Claude Codeが出してくる設計をそのまま使うんじゃなくて、「このライブラリは使わない方がいい」「エラー時はこうしてほしい」みたいに方向性を決める。ここが人間の仕事。

Before/After

あるスクレイピング案件の場合：

自分で全部書いた場合の見積もり: 8〜10時間
Claude Codeと一緒にやった実績: 2時間半

内訳はこんな感じ：

  • 要件整理・設計レビュー: 30分（自分）
  • コーディング: 40分（Claude Code）
  • テスト・デバッグ: 50分（自分 + Claude Code）
  • ドキュメント作成: 30分（Claude Code）

8割くらいはClaude Codeがやってくれるけど、残り2割は自分がやらないとダメ。特にテストは絶対自分の目で確認する。クライアントに納品するものだから。

つまずいたこと

Claude Codeに丸投げして失敗したことがある。

スクレイピングの案件で、Claude Codeが書いたコードがサイトの利用規約に違反するやり方だった。robots.txtを無視してたし、リクエスト間隔も短すぎた。

これは自分がちゃんとレビューしてなかったのが悪い。Claude Codeは「動くコードを書く」のは得意だけど、「法的にOKか」「相手のサーバーに負荷かけすぎないか」みたいな判断は人間がやる必要がある。

もう一つ。クライアントの要件が曖昧なまま実装を始めて、納品直前に「思ってたのと違う」って言われたことがある。これはClaude Codeの問題じゃなくて、自分のヒアリング不足。

Claude Codeは最強の実装パートナーだけど、要件定義とレビューは人間の責任。

ステップ4: テストと納品

テストは3段階でやってる：

  1. 動作テスト: そもそも動くか。エラーが出ないか
  2. エッジケーステスト: データが空の場合、文字化けした場合、ネットワークエラーの場合
  3. クライアント環境テスト: クライアントのPCで動くか。Pythonのバージョン違いとか

3番目が意外とハマる。自分の環境では動くのに、クライアントの環境では動かないパターン。これを防ぐために、最近は仮想環境（venv）ごと納品するか、Docker化して渡すようにしてる。

納品物は毎回こんな構成：

project/
├── README.md          ← 使い方マニュアル
├── requirements.txt   ← 依存ライブラリ一覧
├── setup.sh          ← セットアップスクリプト
├── src/              ← ソースコード
├── tests/            ← テストコード
└── docs/             ← 追加ドキュメント

README.mdは特に丁寧に書く。クライアントが非エンジニアのことも多いから、「ターミナルを開いて、このコマンドを打ってください」レベルで書く。ここもClaude Codeに書かせて、自分が読み直す。

ステップ5: お金の話

正直に書く。

初月の実績：
受注件数: 4件
合計売上: 約55,000円
作業時間: 合計約15時間
時給換算: 約3,600円

2ヶ月目：
受注件数: 6件
合計売上: 約105,000円
作業時間: 合計約20時間
時給換算: 約5,250円

2ヶ月目に売上が倍近く増えたのは、リピーターがついたのと、提案の通過率が上がったから。

クラウドワークスの手数料は20%取られるので、実際の手取りはここから2割引いた額。

大学の授業がある日は1〜2時間、休日は3〜4時間くらい作業してる。

全体を通して学んだこと

3つある。

1. Claude Codeは「実装」の工数を8割減らしてくれる。でも「要件定義」と「品質保証」は減らない。

コードを書く時間は劇的に短くなった。でも、クライアントが本当に欲しいものを聞き出す時間と、納品物が本当に動くか確認する時間は変わらない。むしろ、実装が速くなった分、ここに時間をかけられるようになった。

2. 「なんでもできます」より「これが得意です」の方が受注できる。

最初は何でもやろうとしてたけど、「データ収集・整理の自動化」に絞ってからは受注率が上がった。クライアントは専門家に頼みたい。

3. AIで開発してることは、わざわざ言わなくていい。

クライアントが求めてるのは「動くもの」「期日通り」「丁寧な対応」。開発手法は聞かれない限り言わなくていい。

次にやること

今は1件あたりの単価が低め（5,000〜30,000円）なので、もう少し大きい案件にチャレンジしたい。具体的には：

  • Webアプリ開発（10万円〜）
  • 定期的なメンテナンス契約（月額制）
  • 自分のツールをパッケージ化して販売
