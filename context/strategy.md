# 収益化戦略ノウハウ

## ジャンル選定の3要素（スコアリング基準）

記事のトピックを選定する際、以下の3つの掛け算でスコアリングする。

### 1. 市場の大きさ（1〜5点）
- 5点: X/noteで関連キーワードの発信者が多数、フォロワー数万人規模のアカウントが複数
- 4点: 一定の発信者がいる、関心層は広い
- 3点: ニッチだが確実に需要がある
- 2点: 関心層が限定的
- 1点: ほとんど需要が見込めない

### 2. 悩みの深さ（1〜5点）— HARM法則ベース
- 5点: Money（稼ぎたい）に直結。お金を払ってでも知りたい
- 4点: Ambition（キャリア・スキルアップ）に直結
- 3点: HARMに間接的に関連
- 2点: 「知ってると便利」レベル
- 1点: 純粋な好奇心・雑学

### 3. 悩みの緊急度（1〜5点）
- 5点: 今すぐ知らないと損する（新機能リリース直後、期間限定）
- 4点: 近い将来必要になる
- 3点: いつか必要になるかも
- 2点: 急ぎではない
- 1点: 完全に任意

### 4. 差別化可能性（1〜5点）
- 5点: 誰も書いてない＋自分の実体験で語れる
- 4点: 競合は少ない＋独自の切り口がある
- 3点: 競合はいるが差別化できる角度がある
- 2点: 競合が多く差別化が難しい
- 1点: 既に良質な記事が飽和

### 5. 過去実績との相性（1〜5点）
- 5点: 過去にスキ数が多かったテーマと同系統
- 4点: 過去記事と関連性が高い（シリーズ化できる）
- 3点: 新しいテーマだが読者層は重なる
- 2点: 読者層が少しずれる
- 1点: 全く新しい方向

## 有料記事の価格設定

| 記事タイプ | 推奨価格 | 条件 |
|---|---|---|
| 速報（新機能レビュー） | 500円 | リリース48時間以内の一次情報 |
| 実録（案件記録） | 1,500〜2,000円 | コード・プロンプト全文付き |
| ノウハウ（完全ガイド） | 1,000〜2,000円 | テンプレート・設定ファイル付き |
| 入門記事 | 0円（無料） | 集客・ブランディング目的 |

## 売上推測の公式
- スキの数 × 記事の価格 × 1.5 ≒ おおよその売上
- スキ100以上の有料記事 = 「売れている」

## 有料記事の無料/有料エリア設計

### 無料エリア（冒頭30〜40%）
- 記事の価値が伝わる導入
- 「何が得られるか」のリスト
- 実績・Before/Afterの一部
- 無料エリアだけでも「役に立った」と感じさせる

### 有料→無料境界に入れるもの
- 「この先で手に入るもの」リスト（3〜5項目）
- 「なぜ有料なのか」の理由（外注相場との比較等）

### 有料エリア（残り60〜70%）
- 実際のコード/プロンプト全文
- 具体的な数字・設定ファイル
- テンプレート・チェックリスト
- カスタマイズ方法・応用例

## PDCAサイクル
1. 記事投稿
2. 1週間後にPV・スキ数・売上を記録（performance.md）
3. 反応が良かったテーマ・構成を分析
4. 次の記事に反映（Phase 2のスコアリングに使う）

## メインコンテンツ戦略（Claude / Anthropic 特化）

### コンセプト
「Claudeのことならこの人」— Claude / Claude Code の実務活用における第一人者ポジションを取る。

### 扱うトピック（Claude特化）
- Claude Codeの新機能・アップデート速報
- Claude Code の実務活用テクニック（スキル設計、エージェント構築、MCP活用等）
- Claude Dispatch / Cowork / Desktop の使い方
- Claude API / SDK の活用法
- Anthropic の公式ブログ・発表の解説
- Claude の料金プラン比較・選び方
- Claude vs 他ツール（ChatGPT, Gemini等）の比較 ← Claude視点で書く

### 扱わないトピック
- ChatGPT / Gemini / その他AIツール単体の話題
- AI全般のニュース（Claude関連でない限り）
- プログラミング一般論（Claudeと絡めない限り）

### 収益モデル
- Claude新機能リリース → 「最強の使い方」を速攻で有料500円
- Claude Code活用ガイド → 有料1,000〜2,000円（テンプレート・設定ファイル付き）
- 入門記事・比較記事 → 無料（集客・ブランディング）
- 実録・ノウハウは週1でストック型コンテンツとして蓄積
