# /article-edit スキル

生成済みの草稿を編集して、noteに投稿できる状態に仕上げるスキル。

## 起動時の入力

ユーザーに以下を確認する:
1. 編集対象の草稿ファイルパス（drafts/内のファイル）

## 実行フロー

### Phase 1: 草稿の読み込み・分析
- 対象ファイルを読み込む
- 文字数・構成・トーンを分析
- `context/voice-samples.md` と比較して文体の一貫性を確認

### Phase 2: 7スウィープ編集
以下の7つの観点で順番にチェック・修正する:

1. **明確さ**: 曖昧な表現を具体的にする
2. **声**: AI感のある表現を自然な大学生の言葉に置き換える
3. **So What?**: 各段落に「だから何？」の答えがあるか確認
4. **根拠**: 主張に数字・事例・ソースがあるか確認
5. **具体性**: 抽象的な説明を具体例に置き換える
6. **感情**: 読者の感情を動かすポイントがあるか確認
7. **リスク除去**: 誤情報・誇張・クライアント情報の漏洩がないか確認

### Phase 3: SEO最適化
- タイトルに検索されやすいキーワードが入っているか
- 見出しが内容を正確に要約しているか
- 冒頭100文字で記事の価値が伝わるか

### Phase 4: note向け最終整形
- 表をリスト形式に変換
- コードブロックをインデント付きテキストに変換
- 装飾は見出し（##, ###）、太字（**）、リスト（-）のみ
- 最終文字数を表示

### Phase 5: 保存
- 編集済みファイルを上書き保存
- メタ情報の `status` を `edited` に更新
- 変更サマリーを表示
