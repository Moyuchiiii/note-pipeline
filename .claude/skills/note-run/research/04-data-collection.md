# データ収集・記録（note記事調査版）

## 目的

検索で得られた情報を構造化して記録する。このファイルで定義するフォーマットは、
Phase 5（検証）の入力データ、および記事執筆の素材となる。

## 出力ファイル

`drafts/research/research_{日付}_{トピック}.md` に以下のフォーマットで記録する。

## フォーマット

### ファイル冒頭

```markdown
# [トピック名] 調査記録 YYYY-MM-DD

## 調査概要
（Phase 1-3 で決定したトピック情報をここに記載）

## 調査ログ（全クエリ一覧）

| # | 検索クエリ | 言語 | ヒット数 | 採用数 | 適応拡張 |
|---|-----------|------|---------|--------|---------|
| 1 | "Claude Code pricing 2026" | EN | 5件 | 3件 | — |
| 2 | "Claude Code 料金 変更" | JA | 8件 | 2件 | — |
| 3 | "Anthropic plan update latest" | EN | 3件 | 2件 | 適応拡張 |

（使用した検索クエリをすべて記録する。省略禁止。）
```

### 情報別セクション

```markdown
## 収集した情報

### [トピック・情報のタイトル]

- **[記事タイトル/トピック]**
  - ソース: [記事タイトル](URL)
  - 取得日: YYYY-MM-DD
  - 鮮度: [1週間以内 / 2週間以内 / 1ヶ月以内 / 3ヶ月以内 / それ以上]
  - 信頼度: ✅ 一次データ / 🔶 二次データ / ⚠️ 参考情報
  - 検証状態: [未検証] / [検証済]（05-verification.md で更新）
  - 内容（全文）:
    ここに記事やソースから取得した情報をそのまま記録する。
    要約や省略はしない。取得した内容を原文のまま貼り付ける。
    長文でも省略せず全文を記載すること。

## ソース不明の情報（信頼性低）

- **[トピック]**
  - ソース: なし
  - 信頼度: ⚠️ ソースなし（信頼に欠ける情報）
  - 内容: ～～～
  - 備考: SNSで言及を確認したがソース元記事が見つからなかった 等

## 不採用にした情報

- [記事タイトル](URL) — 不採用理由: 古い情報（2024年の記事）
- [トピック](ソースなし) — 不採用理由: ソース不明

## 信頼度サマリー

- 情報充実度: ★☆☆☆☆〜★★★★★
- ソースの質: 一次データN件、二次データN件、参考情報N件
- 鮮度: [主要な情報の鮮度帯]
```

## 記録ルール（厳守）

### 絶対ルール
- 検索クエリは使ったもの **全件** を記録する。1つも省略しない
- 収集した情報は **要約・省略せず**、取得した内容をそのまま記録する
- 「...」「以下省略」「詳細は割愛」等の **省略表現は禁止**
- 「特になし」と記載する場合も、どのクエリで検索して該当がなかったかを記録する
- **すべての情報にソースURL（出典元リンク）を付記する**
- ソースが確認できない情報は「⚠️ ソースなし」として **分離記録する**

### 数値データの記録
- 価格・ユーザー数・売上等の数値には、**必ず出典年と出所を記載する**
- 例: 「月額$20（Anthropic公式, 2026年3月）」

### 情報の分類タグ
- `[事実]` — 確認された事実（ソースあり）
- `[分析]` — ソースの著者による分析・見解
- `[推論]` — 調査者（AI）による推論。根拠を明記
- `[未検証]` — 単一ソースのみで確認。05-verification.md で検証予定
- `[検証済]` — 2つ以上の独立ソースで確認済み
