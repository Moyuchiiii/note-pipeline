# 検索戦略（note記事調査版）

## 目的

Phase 1-3で決定したトピック情報に基づき、記事執筆に必要な情報を効率よく収集するための
検索クエリを設計・実行する。

## 検索クエリ生成ルール

### クエリバリエーション

トピックについて **最低2〜3個のクエリバリエーション** を生成する：

- **言い換え**: 同じ内容を別の表現で検索する
- **具体化**: 固有名詞・数字・年号を含むクエリ
- **拡張**: 関連キーワードを含む広いクエリ

```
例: 「Claude Codeの料金変更は？」
→ "Claude Code pricing 2026"
→ "Claude Code plan update latest"
→ "Anthropic Claude Code 料金 変更 2026"
```

## 多言語検索ルール

| 条件 | 検索言語 |
|------|---------|
| 英語圏の情報・グローバルなAIトピック | 英語のみ |
| 日本向けの話題・活用事例 | 日本語メイン + 英語補助 |

- 英語で有益な情報が見つかった場合、記録は日本語に翻訳する

### 言語別検索テンプレート

```
英語: "[topic] [year]"
日本語: "[トピック] [年] [キーワード]"
```

## 情報ソース優先度

| 優先度 | ソースの種類 | 例 |
|--------|-----------|---|
| 最高 | 公式情報・一次ソース | Anthropic公式ブログ、Claudeリリースノート |
| 高 | 大手テックメディア | TechCrunch、The Verge、Gigazine |
| 中 | 専門ブログ・Zenn・Qiita | 技術者の実録、実践記事 |
| 低 | SNS・個人ブログ | X/Twitter、はてなブログ |

- 低優先度ソースでも、具体的な実例・スクリーンショット付きの情報は有用
- ソースの種類は収集結果ファイルに明記すること

## 鮮度重み付け（Freshness Weighting）

| 鮮度 | 優先度 | 扱い |
|------|--------|------|
| 1週間以内 | 最優先 | 記事の中心情報 |
| 2週間以内 | 高 | 主要な情報として扱う |
| 1ヶ月以内 | 標準 | 通常通り記載 |
| 3ヶ月以内 | 低 | 背景情報として使用。日付を明記 |
| 3ヶ月超 | 最低 | 原則使用しない。使う場合は「⚠️ 古い情報」と明記 |

## 適応的クエリ拡張（gpt-researcher方式）

初回検索の結果を見て、**追加クエリを動的に生成する**：

1. **情報不足の軸**: 初回検索でヒットが少なかった観点について、クエリを言い換えて再検索
2. **新発見の深掘り**: 検索中に見つかった重要キーワードで追加検索
3. **反証検索**: 見つかった主要な主張に対して、反対意見・批判を検索する

```
適応拡張の判断基準:
- ヒット0件 → クエリを大幅に変えて再検索（必須）
- ヒット1-2件 → クエリを微調整して追加検索（推奨）
- ヒット3件以上 → 十分。次の観点に進む
- 重要な新情報発見 → 関連クエリを追加（推奨）
```

## 検索の終了条件

- トピックの全サブ質問（Phase 1-3のトピック情報から抽出したもの）を検索済み
- 適応拡張を1ラウンド実施済み
- 各サブ質問に対して最低2件の情報を収集済み（0件の場合は「該当なし」と記録）

## 出力形式

検索が終わったら、以下の2点をまとめる：

1. **検索クエリリスト** — 使ったクエリを全件記録（省略禁止）
2. **調査軸メモ** — 「何について / どんな観点で / どの方向から調べたか」を箇条書き

これらを `drafts/research/research_{日付}_{トピック}.md` に記録する。

## MCP連携による検索強化

### deep-research スキル（firecrawl + exa）

通常のWeb検索に加えて、deep-researchスキルのMCPツールを活用できる:

- **firecrawl MCP**: Webページの全文スクレイピング。検索結果のURLから詳細な情報を取得する際に使用
- **exa MCP**: Neural search。通常のキーワード検索では見つからない意味的に関連する情報を発見

### 使い分け

| 状況 | 使うツール | 理由 |
|------|----------|------|
| キーワードが明確 | 通常のWeb検索 | 速い・確実 |
| 類似事例・関連情報を広く探す | exa Neural search | 意味的検索で未知の情報を発見 |
| 記事の全文を取得したい | firecrawl | ページ全体をクリーンにスクレイピング |
| 学術論文・技術文書 | exa | 学術・技術コンテンツに強い |

### exa-search の活用手順

```
1. 通常検索で概要を把握
2. 情報が不足している観点について exa で補完検索
3. 特定の記事URLが見つかったら firecrawl で全文取得
```

### 検索ツールの優先順位

1. **Web検索**（デフォルト）: まず通常検索で情報を集める
2. **exa Neural search**（補完）: 通常検索で不足した場合に意味的検索で補完
3. **firecrawl**（深掘り）: 重要な記事の全文を取得する場合

⚠️ MCPツールが利用できない環境では、通常のWeb検索のみで実行する。
